Embedded systems?

Everything that runs code but does not look like a computer

  • In-car navigation
  • MP3 players
  • Medical devices
  • IoT devices
  • ...
So what is embedded? Loosely defined concept... I think of it as... Examples are... Too many to sum up here. Common: * Constrained * Special purpose * Safety critical * Hard to update TinyGo focuses on a more specific category: microcontrollers.

Microcontrollers

CPU 10MHz-100MHz
RAM 2kB-100kB
Flash 10kB-1MB
Architecture ARM, AVR, RISC-V, (many others)
The kind of MCUs that TinyGo focuses on. Executed directly from flash. Most fall in this range.

Why Go on Embedded?

  • Security[1]
  • Ease of programming
  • Concurrency
  • Testability
Security: Embedded means C 2/3 security problems are memory safety issues (OOB, double free, etc) IoT means more connected devices Ease of programming: Cross compiling is a pain Embedded toolchains are a mess Lack of standards - Arduino fixes all of this (hence popular) Concurrency RTOS replacement tasks (threading) + mutexes etc memory management HAL network stack DMA Testability: I'll get to that...
